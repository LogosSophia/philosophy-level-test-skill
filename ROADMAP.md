# 路线图 / Roadmap

本路线图用于规划 `哲学水平测试.skill` 后续发展方向。

---

## v0.1：文本版 skill

状态：已完成。

内容：

- README 中英双语介绍
- 核心 skill 主体
- 评分规则
- 术语表
- 示例
- prompt 模板
- 快速开始文档

目标：

让用户可以直接复制 skill 到任意 AI 工具中使用。

---

## v0.2：示例库扩展

目标：补充更多测试样例，尤其是高频伪深刻句式。

计划样例类型：

- 自由类
- 意识类
- 宇宙本体类
- 历史终局类
- 文明命运类
- 语言边界类
- 爱/意义/自我类
- 玄学表达类
- 社会评论类
- AI 生成哲学段落类

---

## v0.3：追问模式强化

目标：从“一次性打分”扩展到“三轮追问后评分”。

流程：

1. 提取核心命题
2. 提出 3 个致命追问
3. 用户回答
4. 再次审查
5. 给最终评分

用途：

- 更适合严肃哲学讨论
- 更能测试文本是否有继续推进能力
- 避免只凭第一段话误判

---

## v0.4：网页 Demo

目标：做一个可试玩网页。

基本功能：

- 输入哲学观点
- 点击“开始测试”
- 输出评分、段位、伪深刻指数、最大漏洞、致命追问

可选技术：

- Streamlit
- Hugging Face Spaces
- Vercel
- Cloudflare Pages
- GitHub Pages + API

---

## v0.5：中英文双版本 skill

目标：从中文主规则扩展为英文完整版本。

文件结构：

```text
philosophy_level_test.skill.zh.md
philosophy_level_test.skill.en.md
```

注意：

英文版需要保持术语稳定，不要让 Illegal Synthesis、Invalid Position、Scale Drift 等术语漂移。

---

## v1.0：稳定版

目标：形成稳定、可传播、可调用的哲学水平测试系统。

应具备：

- 完整 skill 规则
- 稳定评分表
- 20+ 示例
- 中英术语表
- 可试玩 demo
- 清晰授权说明
- 贡献指南
- 基础测试集

---

## 长期方向

1. **哲学压力测试器**：从单段文本评测扩展到完整理论草稿评测。
2. **伪深刻检测器**：专门识别概念烟雾、宏大空话、玄学包装。
3. **理论审稿助手**：用于论文草稿、体系草稿、自创理论的前提审查。
4. **互动哲学面试官**：通过多轮追问测试思想承压能力。
5. **公共测试榜单**：收集高分文本、低分典型文本和经典漏洞案例。
