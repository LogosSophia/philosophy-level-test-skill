# 贡献指南 / Contributing

欢迎提交改进建议、示例文本、评分校正和规则修订。

This project welcomes suggestions, examples, scoring corrections, and rule refinements.

---

## 可以贡献什么？

你可以贡献：

1. 新测试样例
2. 更好的评分边界
3. 更准确的术语翻译
4. 某个审查项的反例
5. 某类伪深刻文本的样本
6. 对非法综合、位置非法、尺度漂移等规则的补充
7. README 或传播文案优化

---

## 提交原则

请尽量遵守以下原则：

1. **批判文本，不攻击作者。**
2. **不要把“读过某哲学家”当成高分依据。**
3. **不要因为语言华丽就给高分。**
4. **不要因为观点反常就直接判低分。**
5. **评分依据应尽量回到对象、位置、尺度、概念和非法综合。**
6. **新增术语需要同步更新 `glossary.md`。**
7. **新增评分维度需要保证总分仍为 100。**

---

## 提交示例时请包含

建议格式：

```markdown
## 输入文本

```text
这里粘贴待测试文本。
```

## 建议评分

- 哲学水平评分：__/100
- 伪深刻指数：__/100
- 段位：____

## 主要理由

- 最大漏洞：____
- 致命追问：____
- 修改建议：____
```

---

## 修改规则时请说明

如果你修改 skill 主体规则，请说明：

1. 修改了哪个审查项。
2. 为什么原规则不够。
3. 新规则会影响哪些评分。
4. 是否会导致 AI 执行更稳定。
5. 是否需要同步修改 `scoring.md`、`glossary.md` 或 `examples.md`。

---

## 风格原则

这个项目的风格是：

- 外层娱乐化
- 内核严肃化
- 输出可读
- 评分严格
- 批判犀利但不羞辱人

可以毒舌，但必须准确。
可以好玩，但不能胡判。
